# Team Divider - 更新ログ

## v1.00（2025-03-26）

リファクタリング・バグ修正を経た初回安定版。

### バグ修正

- **リロール時に待機者が消失する問題を修正**
  - チェックOFF でリロールを繰り返すと、枠不足で待機になった人が specs から消えて二度と戻れなくなるバグを修正
  - specs の算出を「プールからのスライス」ではなく「チームに入らなかった全員」から算出するよう変更
- **リロール時に★優先参加者が待機者に落ちる問題を修正**
  - シャッフル後の先頭取得で優先者が後ろに回る可能性があったため、優先者を先にチーム枠に確保するよう変更

### リファクタリング

- 変数名・関数名を明確化
  - `pid` → `nextPlayerId`, `e()` → `escapeHtml()`, `prow()` → `renderPlayerRow()` 等
  - `balanceRC()` → `balanceByRankCount()`, `stScore()` → `calcSameTeamScore()` 等
- ヘルパー関数の追加（`findPlayer()`, `getSelectedMode()`, `showRerollControls()`）
- 選出優先度ソートを `sortBySelectionPriority()` に分離
- 同レシオスワップ最適化を `optimizeSameRankSwaps()` に分離
- `sort(() => Math.random() - 0.5)` → Fisher-Yates シャッフルに置換
- セクションコメント・JSDoc コメントの整備

### その他

- レシオ表示を「初・中・上」から「★・★★・★★★」に変更
- ヘッダーにバージョン表記（v1.00）を追加
- ページタイトルを `TEAM DIVIDER v1.00` に変更
- リロールチェックボックスのデフォルトを OFF に変更

## v1.00 以前の開発履歴

初回リリースに至るまでの開発経緯。

### 初期実装

- 参加者登録（最大 20 人、レシオ 3 段階）
- 5v5 チーム自動生成（スネークドラフト方式）
- 待機者ローテーション・優先参加フラグ
- 過去ラウンドの同チームペア重複を最小化する最適化ループ
- ラウンド履歴表示

### Bootstrap 5 対応・待機者ローテバグ修正

- Bootstrap 5.3.3 を導入
- 待機者が同じ人に偏るバグを修正（登録順バイアス）
  - ソート条件に累計待機回数を追加

### UI 改善・機能追加

- 白背景（#FFF）ベースの配色に変更
- 各プレイヤーの参加数・待機数を表示
- ラウンド履歴に待機者・使用モードを追加
- チーム分けモードの追加
  - 「均等レシオ分け」を標準モードとして追加（各レシオの人数を均等に分配）
  - 既存のスネークドラフト方式を「レシオ合計均等」オプションに変更
- 👁待機機能を追加（今ラウンドを欠席し待機者になる）

### リロール機能

- ラウンド生成後にチーム割り当てを再抽選するリロール機能を追加
- リロール回数の表示（表示部・履歴の両方）

### データ永続化

- localStorage による参加者データの自動保存・復元
- 保存データ削除ボタンの追加

### リロール改善

- リロール時にモード切替を可能に（生成時と別モードでリロール）
- 枠不足の待機者をリロールに含めるチェックボックスを追加
- 登録順バイアスの除去（ソート最終段にランダムタイブレーカー追加）

### 表記統一

- 「休憩」の文言を全て「待機」に統一
