# Team Divider - 更新ログ

## v1.12（2026-04-29）

ドラッグ＆ドロップ手動入替機能の追加と、バランス優先時の選出ロジック改善。

### 機能追加

- **チーム⇔待機者のドラッグ＆ドロップ入替**
  - ラウンド生成後、BLUE / RED / 待機者 のプレイヤーをドラッグ操作で入替可能
  - プレイヤーを別のプレイヤーにドロップすると ID をスワップ（チーム人数は自動維持）
  - 視覚フィードバック: ドラッグ中はターゲットを破線アウトラインでハイライト
  - 入替後に重複ラウンド検出を再実行（過去と同じ組み合わせなら警告）

### 機能追加（追補）

- **選出傾向に「なし」オプションを追加（デフォルト）**
  - 低レシオ優先・高レシオ優先のどちらにも偏らない、ニュートラルな選出
  - バイアス補正がゼロになり、構成選択は参加回数バランス調整のみに従う
  - 新規ラウンド生成のデフォルトを「なし」に変更

### バグ修正

- **バランス優先時に参加回数が考慮されない問題を修正**
  - レシオグループ内ソートが `getLastPlayedIndex` のみで、参加回数の少ない人を優先できていなかった
  - 新しい優先順位: ①前ラウンド待機者 → ②参加回数が少ない順 → ③待機回数が多い順 → ④最後の参加が古い順 → ⑤ランダム
  - グループ内の選択肢がある場面（在籍人数 > 構成数）で、待機が偏らないようになる
- **構成選択（レシオ人数の組み合わせ）に参加回数バランス調整を追加**
  - 各レシオの平均参加回数を算出し、平均より参加が多いレシオを多く含む構成にペナルティ
  - 例: レシオ1 在籍5人で全員 27回 / レシオ2 在籍8人で 17回の場合、
    従来は (4,4,2) が約75%選ばれていたが、調整後は (2,6,2) が優先される
  - 参加回数が全員均等な場合は playFactor=1 で影響なし
  - バイアス設定（低/高レシオ優先）は引き続き有効、補正として作用

## v1.11（2026-04-28）

ロックグループ機能の追加。

### 機能追加

- **ロックグループ（同チーム強制）**
  - 各プレイヤーに 🔒A / 🔒B / 🔒C のロックグループを設定可能
  - 同じグループに所属するプレイヤーは必ず同じチームに配属される
  - 参加者リストの 🔒 ボタンクリックで なし→A→B→C→なし のサイクル変更
  - チーム表示にロックバッジを表示
  - グループごとに色分け（A=パープル / B=ティール / C=オレンジ）
  - localStorage 復元時の後方互換あり（既存データの lockGroup は 0 として扱う）

### アルゴリズム

- 既存のチーム分けアルゴリズム（均等レシオ分け / レシオ合計均等）の後段に
  `consolidateLockGroups` を追加し、分散したロックメンバーを多数派側に寄せる
- スワップ最適化（`optimizeSameRankSwaps` / Pass 2）に
  ロックグループ分断防止ガード（`wouldSplitLockGroup`）を追加
- 寄せきれない場合（グループ人数 > 5 等）はトーストで警告

## v1.10（2026-03-27）

汎用化・選出ロジック強化・UI改善。

### 機能追加

- **選出方式オプション**
  - バランス優先（デフォルト）: レシオの均衡を重視し、連戦を許可
  - ローテーション優先: 全員が均等に参加することを重視
- **選出傾向オプション（バランス優先時）**
  - 低レシオ優先（デフォルト）/ 高レシオ優先の2択
  - 同スコアの構成間でバイアス付きランダム選択
- **各レシオから最低1名選出オプション（デフォルトON）**
  - バランス優先・ローテーション優先の両方で有効
  - 候補なしの場合は自動フォールバック
- **重複ラウンド検出**
  - 過去ラウンドと同じチーム構成（Blue/Red逆転含む）を検出し、バッジ＋トーストで警告
- **レシオのクリック変更**
  - 参加者リストのレシオバッジをクリックで 1→2→3→1 とサイクル変更
- **レシオ別人数の表示**
  - 参加者リストのヘッダーにレシオ別の人数バッジを表示

### バグ修正

- **バランス優先時にレシオ差が生まれる問題を修正**
  - 構成の全列挙＋スコアリングでレシオ合計が偶数になる構成を選出
  - 均等レシオ分け時は各レシオの人数が偶数になる構成を優先
- **スネークドラフトでレシオ差が蓄積する問題を修正**
  - 2パス構成に変更: Pass1でレシオ差を最小化、Pass2でレシオ差を維持しつつ同チームペアを最小化
- **リロール時に選出方式・チーム分けモードが反映されない問題を修正**
  - チェックON時は generateRound と同じ選出ロジックを通すよう変更
- **レシオ合計の表示値が内部値（0,1,2）で計算されていた問題を修正**
  - 表示用の calcRankSum で rank+1 を使用するよう変更
- **HTMLのdivタグ不整合によるレイアウト崩れを修正**

### 表記変更

- LoL 関連の表記を全て削除（汎用ツール化）
- 「ランク」→「レシオ」に統一
- レシオ表示を「初・中・上」→「1・2・3」に変更（バッジを丸型に統一）
- 「観戦」→「待機」に統一
- ファイル名を `lol-team-divider.html` → `team-divider.html` に変更
- ヘッダー・タイトルを `TEAM DIVIDER` に変更

### UI改善

- 参加者登録のレシオボタンに「レシオ」ラベルを追加
- `.player-scroll` / `.history-scroll` をビューポート基準の高さに変更
- 選出方式がローテーション時、選出傾向オプションを自動非表示
- リロールボタンとチェックボックスをグループ化し視覚的に紐付け
- ラウンド履歴に参加者のレシオバッジとチームレシオ合計を表示

### 技術ノート

- 内部ではレシオを `0, 1, 2` で管理（初級/中級/上級時代の名残）
- 表示上は `1, 2, 3` に変換、合計値の計算には `rank + 1` を使用
- ソースコード内のコメントにも注記を追加

## v1.00（2026-03-27）

リファクタリング・バグ修正を経た初回安定版。

### バグ修正

- **リロール時に待機者が消失する問題を修正**
  - チェックOFF でリロールを繰り返すと、枠不足で待機になった人が specs から消えて二度と戻れなくなるバグを修正
  - specs の算出を「プールからのスライス」ではなく「チームに入らなかった全員」から算出するよう変更
- **リロール時に★優先参加者が待機者に落ちる問題を修正**
  - シャッフル後の先頭取得で優先者が後ろに回る可能性があったため、優先者を先にチーム枠に確保するよう変更

### リファクタリング

- 変数名・関数名を明確化
  - `pid` → `nextPlayerId`, `e()` → `escapeHtml()`, `prow()` → `renderPlayerRow()` 等
  - `balanceRC()` → `balanceByRankCount()`, `stScore()` → `calcSameTeamScore()` 等
- ヘルパー関数の追加（`findPlayer()`, `getSelectedMode()`, `showRerollControls()`）
- 選出優先度ソートを `sortBySelectionPriority()` に分離
- 同レシオスワップ最適化を `optimizeSameRankSwaps()` に分離
- `sort(() => Math.random() - 0.5)` → Fisher-Yates シャッフルに置換
- セクションコメント・JSDoc コメントの整備

### その他

- ヘッダーにバージョン表記を追加
- リロールチェックボックスのデフォルトを OFF に変更
